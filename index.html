Safety Gate & Review Criteria

Verify constraints protecting configuration settings, credentials, and production services

Zero-Exposure Secret Policy

To prevent security leaks, the Hermes operating model absolutely forbids logging, writing, printing, staging, or committing secret credentials. Check local profiles, and reference status without copying values.

SSH Keys & Auth

Never copy private SSH keys or path configurations under ~/.ssh/.

Environment Secrets

API keys, Discord tokens, and .env contents must remain local.

auth.json configs

Local credentials and API token JSON files are untracked.

Provider Tokens

Do not copy raw output files containing provider connection strings.

Review-Required Criteria Gated Actions

Code & Script Modifications

Edits to shell scripts (bin/hermes-remote), tools, or application code.

Remote System Configurations

Edits to launchd services, systemd, or ~/.hermes/config.yaml profiles.

Gateway State Restarts

Starting/stopping the background Gateway dispatcher service or changing ports.

Key / Credentials Updates

Proposing permission grants, auth configurations, or API credential updates.

Recurring Analysis Automation

Defining recurring Cron jobs or automatic dispatch intervals in Kanban dispatcher.

How to Inspect Results

Standard instructions and commands to verify workspace changes and promote tasks

1

Check Isolated Git Diff

Inspect exactly what changed in the current isolated worktree to prevent extraneous staging.

2

Run Script Syntax Tests

Check bash scripts using syntax linting before launching remote operations.

3

Validate Research Outputs

Ensure brief files, source ledgers (.jsonl), notes, and reports are present in their corresponding paths.

4

Check SSH Target Status

Confirm target connectivity using the remote manager script commands.

verification_workflow.sh Terminal Presets 
# 1. Verify changes in the isolated workspace
git status --short
git diff --stat

# 2. Syntax check operations script
bash -n bin/hermes-remote

# 3. Check connectivity & gateway status on remote target
bin/hermes-remote --target bobeen-mac check-ssh
bin/hermes-remote --target bobeen-mac status

# 4. Verify research files existence
test -f research/briefs/<YYYY-MM-DD>-<slug>.md
test -f research/sources/<YYYY-MM-DD>-<slug>.jsonl
test -f reports/<YYYY-MM-DD>-<slug>.md
