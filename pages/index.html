Hermes Workspace

Remote Hermes Agent가 작업을 계획하고, 실행하고, 결과물을 남기는 git-backed workspace

작업 요청 예시 보기 결과물 확인 방법 보기

What Hermes Can Do

ops-change

원격 시스템 설정 변경, config 수정, 스크립트 실행 등 운영 작업

remote-config

원격 Mac의 Hermes 설정, launchd 서비스, MCP 서버 관리

incident-triage

Discord 스레드 분석, gateway 로그 확인, 문제 진단 및 해결

market-research

웹 검색, 시장 조사, 가격 비교, 정책/법률 검증

analysis-report

데이터 분석, 코드 리뷰, research brief 작성 및 보고서 생성

How To Ask

좋은 요청 프롬프트 구조

role: [Control-side remote ops | Hermes-side workspace 작업]
task type: [ops-change | remote-config | incident-triage | market-research | analysis-report]
작업 내용: [구체적인 요청사항]
결과물 경로: [tasks/ | reports/ | research/]
완료 기준: [done | review-required]

나쁜 요청 / 애매한 요청 예시

❌ "뭐 해줄까?" 너무 일반적이고 목적이 불분명함
❌ "급해, 빨리 해줘" 급함 속성만 있고 작업 범위 없음
❌ "여기요 여기요" 무엇을 원하는지 모호함

프롬프트 카드 5개

hermes-remote check-ssh && hermes-remote status

원격 접속 상태 확인

bin/hermes-remote gateway-restart && log 확인

gateway 재시작 후 로그 확인

market-research로 2026년 AI 모델 가격 비교해줘

최신 시장 가격 조사 요청

incident-triage: Discord 스레드 1512760172660129933 진단해줘

특정 스레드 문제 분석 요청

reports/2026-06-06-hermes-workspace-landing 정리해서 commit 해줘

작업 결과물 정리 및 커밋 요청

Workflow

1 요청 수신 (Discord/CLI)
→
2 task type 판단
→
3 branch/worktree 생성
→
4 작업 수행
→
5 테스트/검증
→
6 결과 보고
→
done review-required

Outputs

경로 설명
tasks/ 작업 스크립트 및 코드 아티팩트
reports/ 작업 결과 보고서
research/briefs/ 시장 조사/분석 brief 문서
research/sources/ 소스 원본 JSONL 원장 (source ledger)
research/notes/ 조사/분석 중 메모
artifacts/ 임시 파일 및 첨부 자료

Review Required

다음 유형의 작업은 반드시 검토 후 merge 필요:

  • 코드 변경
  • 원격 config 변경
  • gateway restart
  • key/auth/secret 관련 작업
  • recurring automation (cron job)

Market Research Mode

web verification 필요

웹 검색을 통한 최신 정보 검증 필수

source ledger 남기기

research/sources/YYYY-MM-DD-slug.jsonl 에 출처 기록

report-only는 done 가능

보고서만 작성하는 경우 검토 없이 완료 가능

Final Report Example

branch/worktree: workspace-landing

changed files: pages/index.html, pages/styles/main.css

result path: pages/index.html

tests/checks: git diff 검증, 정적 HTML 구문 확인

completion mode: review-required (landing page 변경)
